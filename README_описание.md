# Описание работы программы  
Предмет анализа:    
Недвижимость (квартиры, участки)
1.  Задачи  

Сервис чтения данных
  1.  Чтения данных (адрес, стоимость, размер, дата, количество фото, длина описания его вид)
  2.  Приведение к виду (адрес, координаты, стоимость, размер, 
стоимость за единицу размера, дата, первичка вторичка)
  3. Нужно придумать как хранить данные (записывать только при изменении значения)  
  Бэклог: можно смотреть высотность, внутренние дворы (ТЦ, наличие метро, остановки),
  возможно есть ложные объявления нужно отфильтровывать. Срок постройки.  

  Сервис обработки данных  
  Гипотезы по группировке
4. Группировать данные по стоимости и расположению
5. Найти взаимосвяз между данными (построить корреляционную матрицу)
5. Нахождение районов стоимости (кластеров по расположению и стоимости за единицу размера)
6. Нахождение высот стоимости (кластеры по высоте этажей, мб высоты над уровнем моря)
7. При большой квадратуре стоимость квадрата снижается.
8. Зависимость стоимости от срока эксплуатации.

  Сервис хранения данных
8. Принимает в себя данные после этапа придиведения к виду. Имеет колонки с нужными полями.  
9. Хранит эти данные в удобном и компактном виде
10. Отдает данные в сервис анализа. 

Проблема: данные большие, большое количество, особенно по оси времени. Нужно придумать как их эффективно передавать и 
хранить, обрабатывать.

Цель: формировать данные (находить стационарности: по районам, высоте) и находить выбросы, которые и
формируют целевую группу, для изучения.

## Описание сервиса чтения данных
Как я себе представляю это: происходит чтение данных (скраппинг) с Авито (циан и т.д.) все эти данные батчами пишутся 
преобрабатываются и пишутся в Базу данных.  

## Описание сервиса анализа данных
Анализирует данные по параметрам.  
Проверяет гипотезы  
Обучается, пытается восстановить стоимость по параметрам, находит различия (сингулярности)
Можно обучать в несколько потоков, особенно, если будет большой набор данных. (Центральная предельная теорема)

## Описание сервиса хранения данных
Принимает данные, фильтрует, валидирует, пишет в БД. Выдает по запросу в сервис анализа.
